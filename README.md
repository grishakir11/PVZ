# PVZ Moscow Location Optimization

Программная система для выбора и анализа локаций пунктов выдачи заказов на территории Москвы на основе открытых пространственных данных, пешеходной доступности и моделей оптимизационного размещения.

Проект реализует воспроизводимый GIS-pipeline: от подготовки геоданных и построения пешеходного графа до формирования модели спроса, расчёта нескольких конфигураций сети ПВЗ, сравнения результатов и визуализации итогов.

## Назначение проекта

Система предназначена для предварительного пространственного отбора локаций ПВЗ и сравнения различных стратегий развития сети. Она позволяет оценивать не только общегородские показатели, но и распределение эффекта между районами Москвы.

В рамках проекта реализованы:

- подготовка пространственных данных;
- построение пешеходного графа Москвы;
- формирование модели потенциального спроса на основе жилой застройки;
- генерация множества кандидатных локаций;
- модель максимизации охвата;
- модель минимизации среднего времени доступа;
- модель максимизации эффективного спроса;
- ограниченная компромиссная модель;
- сравнение решений по городским и районным метрикам;
- построение карт, графиков и итоговых визуализаций;
- демонстрационный интерфейс на Streamlit.

## Основные модели

В проекте сравниваются четыре модели размещения ПВЗ.

1. **Максимизация охвата**  
   Выбор локаций, обеспечивающих максимальный прирост потенциального спроса в пределах 10 минут пешего доступа.

2. **Минимизация среднего времени доступа**  
   Выбор локаций, которые сокращают среднее взвешенное время пешего доступа до ближайшего ПВЗ.

3. **Максимизация эффективного спроса**  
   Выбор локаций, максимизирующих интегральную полезность сети с учётом постепенного снижения привлекательности ПВЗ при росте времени пути.

4. **Ограниченная компромиссная модель**  
   Вариант, сохраняющий высокий общий результат и дополнительно учитывающий обслуживание удалённых территорий, в частности Зеленограда.

## Установка

    pip install -r requirements.txt
    pip install -r requirements-app.txt

## Запуск демонстрационного интерфейса

    streamlit run app.py

В интерфейсе доступны:

- обзор программной системы;
- выбор модели размещения;
- просмотр выбранных точек на карте;
- сравнение четырёх конфигураций сети;
- просмотр итоговых артефактов;
- формирование отчёта;
- страница запуска полного расчётного pipeline.

## Консольные команды

Проверить структуру проекта:

    python main.py status

Показать найденные CSV-файлы моделей:

    python main.py list-models

Сравнить рассчитанные модели:

    python main.py compare-models

Показать результат отдельной модели:

    python main.py show-model coverage
    python main.py show-model mean_time
    python main.py show-model effective
    python main.py show-model compromise

Сформировать markdown-отчёт:

    python make_demo_report.py

## Режимы работы

### Демонстрационный режим

Использует заранее рассчитанные CSV-файлы и итоговые артефакты. Этот режим предназначен для защиты и быстрого показа результатов без повторного запуска длительных георасчётов.

### Исследовательский режим

Представлен последовательностью скриптов в `scripts/final`. Он охватывает полный расчётный pipeline: подготовку данных, построение графа, формирование спроса, запуск моделей размещения, расчёт метрик и сборку итоговых визуализаций.

## Практический результат

Ключевой результат проекта состоит в том, что разные критерии оптимизации приводят к различным пространственным конфигурациям сети ПВЗ и различным компромиссам между охватом, временем доступа и интегральной полезностью сети.

Наиболее практичным вариантом по итогам исследования является ограниченная компромиссная модель, поскольку она сохраняет высокий общий результат без чрезмерного ухудшения обслуживания удалённых районов города.
