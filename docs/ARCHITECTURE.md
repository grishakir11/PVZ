# Архитектура программной системы

Проект оформлен как программная система поддержки принятия решений для выбора локаций пунктов выдачи заказов на территории Москвы.

## Общая логика

Система реализует воспроизводимый GIS-pipeline:

1. подготовка исходных пространственных данных;
2. построение пешеходного графа;
3. формирование модели потенциального спроса;
4. генерация множества кандидатных локаций;
5. запуск моделей оптимизационного размещения;
6. расчёт городских и районных метрик;
7. визуализация результатов;
8. сравнение конфигураций сети.

## Основные компоненты

- `pvz_system.config` — загрузка конфигурации проекта;
- `pvz_system.data_io` — поиск и чтение рассчитанных CSV-файлов;
- `pvz_system.reporting` — сборка сводок и markdown-отчёта;
- `pvz_system.pipeline_runner` — запуск этапов расчётного pipeline;
- `pvz_system.runtime_paths` — получение путей текущего запуска;
- `pvz_system.ui_path_state` — хранение путей интерфейса;
- `pvz_system.interface` — Streamlit-интерфейс;
- `pages/01_Полный_расчёт_pipeline.py` — страница запуска полного pipeline.

## Режимы работы

### Демонстрационный режим

Использует заранее рассчитанные результаты из `pvz_project/models` и итоговые артефакты из `pvz_project/deliverables`. Предназначен для защиты и быстрого показа результатов.

### Исследовательский режим

Использует скрипты из `scripts/final` и предназначен для полного воспроизведения расчётов.
